## Compare / contrast

Pocock reports using one `/writing-great-skills` skill far beyond writing — for `AGENTS.md`, "docs for
agents," "specs and tickets," and "AFK workflow prompts" — because "structure + leading words +
pruning is useful for any text agents read."

This bundle is built almost entirely of **agent-read text**: `AGENTS.md`, the skill bodies, the rules.
[`rules/skills.md`](../../../../rules/skills.md) governs their *structure* (single-source, thin shims,
portable format, reference-don't-restate) but says nothing about the **prose craft** that makes
agent-read text land — the very "structure + leading words + pruning" Pocock names. So the claim
**extends** `rules/skills.md` with an authoring-craft dimension it doesn't yet capture. His meta-move —
*one* authoring discipline reused across many agent-read artifacts — also mirrors the bundle's own
single-source, author-once philosophy.

A concrete gap this suggests: `rules/skills.md` could gain a **Pattern** on authoring agent-read prose
(lead with the imperative / leading words; structure over paragraphs; prune ruthlessly), so the craft
that makes a skill body effective is captured, not just its wiring.
